## 一、设计理念与原则

### 1.1 设计理念

| 理念 | 说明 |
|------|------|
| **安全可信** | 以蓝色为主色调，传递专业、安全、可信赖的品牌形象 |
| **活力青春** | 橙色作为辅助色，体现大学生群体的活力与朝气 |
| **简洁高效** | 清晰的信息层级，减少认知负担，提升操作效率 |
| **零押金保障** | 通过视觉设计强化"无押金、薪资托管"的核心卖点 |

### 1.2 设计原则

- **一致性**: 三端统一的视觉语言，降低学习成本
- **可用性**: 交互符合用户预期，操作路径最短
- **可访问性**: 支持键盘导航、屏幕阅读器，适配色盲用户
- **性能优先**: 动效轻量化，避免过度渲染

## 六、布局规范

### 6.1 小程序端

```
┌──────────────────────────────┐
│  顶部导航栏（自定义或原生）    │
│  返回按钮 + 标题 + 右侧操作    │
├──────────────────────────────┤
│                              │
│         页面内容区            │
│         全宽，左右边距16px     │
│                              │
├──────────────────────────────┤
│     底部TabBar（固定高度56px） │
│  首页 / 投递 / 消息 / 我的    │
└──────────────────────────────┘
```

**TabBar规范**:
- 高度: 56px
- 图标: 线性图标，尺寸24px
- 文字: 12px
- 选中状态: 主色`#165DFF`
- 未选中状态: 辅助灰`#86909C`

**页面规范**:
- 列表项高度 ≥ 44px（符合微信小程序规范）
- 页面滚动区域使用`scroll-view`
- 下拉刷新使用原生`onPullDownRefresh`
- 上滑加载使用`onReachBottom`

## 七、动效规范

### 7.1 基础动效

| 动效 | 持续时间 | 缓动函数 | 用途 |
|------|----------|----------|------|
| 淡入 | 200ms | `ease-out` | 页面加载、弹窗出现 |
| 淡出 | 200ms | `ease-in` | 页面关闭、弹窗消失 |
| 滑动进入 | 300ms | `ease-out` | 面板展开、列表项出现 |
| 缩放 | 200ms | `ease-out` | 按钮点击、图标缩放 |

### 7.2 业务动效

#### 7.2.1 投递成功动效

```
1. 简历卡片缩小并飞入目标岗位
2. 绿色勾选图标弹出
3. 成功提示文字淡入
4. 整体持续时间: 500ms
```

#### 7.2.2 薪资到账动效

```
1. 金币图标从上方下落
2. 金额数字滚动显示（从0到实际金额）
3. 成功绿背景渐入
4. 整体持续时间: 800ms
```

#### 7.2.3 打卡成功动效

```
1. GPS定位圈收缩
2. 绿色对勾标记出现
3. 位置信息淡入
4. 整体持续时间: 400ms
```

#### 7.2.4 风险预警动效

```
1. 红色闪烁（3次，每次100ms）
2. 震动效果（移动端）
3. 警示文字抖动
4. 整体持续时间: 500ms
```

### 7.3 交互反馈

| 场景 | 反馈方式 |
|------|----------|
| 点击按钮 | 缩放0.95 → 恢复 |
| 表单验证失败 | 输入框红色边框 + 错误提示淡入 |
| 加载中 | 骨架屏 / 加载动画 |
| 空数据 | 空状态插图 + 提示文字 |

## 九、无障碍设计

### 9.1 颜色对比度

| 场景 | 对比度要求 |
|------|------------|
| 正文文字 | ≥ 4.5:1 |
| 大文字（≥18px） | ≥ 3:1 |
| 按钮文字 | ≥ 4.5:1 |

### 9.2 键盘导航

- 所有可交互元素支持Tab键聚焦
- 聚焦状态有明显的视觉反馈（高亮边框）
- Enter键触发主按钮
- Escape键关闭弹窗

### 9.3 屏幕阅读器支持

- 图片必须有alt属性
- 表单元素必须有label关联
- 使用语义化HTML标签
- 动态内容变化时通知屏幕阅读器

### 9.4 色盲友好

- 避免仅用颜色区分状态，配合图标或文字
- 使用色盲安全配色方案

### 10.2 使用说明
